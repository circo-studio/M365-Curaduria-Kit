Cómo funciona la identificación El kit identifica los sitios creados por Teams por su template de SharePoint (GROUP#0, TEAMCHANNEL#0) — esto es universal y no depende del nombre que le ponga el cliente. Los prefijos son opcionales y se usan como criterio adicional si el cliente tiene naming convention propia.

Cómo usar esta pestaña Completar el path donde va a estar el cliente_config.json en la máquina donde se ejecutan los scripts. Los snippets de abajo se actualizan automáticamente.
Flujo de ejecución completo
1
Script_A_Inventario.ps1
Genera el inventario completo del tenant. Exporta CSV con columna Relevar_Documentos. Después completar con SI los sitios a relevar.
2
Script_A_AsignarOwner.ps1
Asigna la cuenta B2B de Circo Studio como SCA en todos los sitios GR- pendientes. Prerequisito de acceso para el Script B.
3
Script_B_Masivo.ps1
Relevamiento completo. Genera CSV por sitio + consolidado. Idempotente — si un CSV previo existe, lo saltea. Puede tardar varias horas.
4
analizar_contenido.py
Procesa todos los CSVs y genera HTML con análisis de actividad post-migración, candidatos a archivo y matriz de destino.
5
generar_informes_sitio.py
Genera un HTML de análisis profundo por cada sitio: duplicados, formatos especiales, archivos grandes, decisiones de migración.
6
generar_indice.py
Genera el HTML índice maestro que enlaza todos los informes. Diferencia prioritarios de automáticos según el config. Punto de entrega al cliente.
↺
Script_C_Refresh.ps1
Actualización mensual incremental. Solo re-procesa sitios con actividad posterior al último CSV. Usar para actualizar el ISI sin volver a relevarlo todo.
Referencia de scripts
PrerequisitoCuenta con rol SharePoint Administrator en el tenant. Completar el cliente_config.json antes de ejecutar.
ParámetroDescripción
-ConfigPathObligatorio. Path al cliente_config.json.
-ForceAuthSwitch. Fuerza re-autenticación aunque haya token cacheado.
-IncludeSubsitesBool. Incluir subsitios. Default: $true.
PrerequisitoScript A Inventario completado. La cuenta B2B de Circo Studio debe estar activa como guest en el tenant.
ParámetroDescripción
-ConfigPathObligatorio. Path al cliente_config.json.
-ForceAuthSwitch. Fuerza re-autenticación.
ImportantePuede tardar varias horas. No cerrar la sesión. Si se interrumpe, volver a ejecutar — los sitios con CSV previo se saltean.
ParámetroDescripción
-ConfigPathObligatorio. Path al cliente_config.json.
-ForceAuthSwitch. Fuerza re-autenticación.
-SoloSitioURL de un único sitio — para debug o reintento puntual.
Lógica de decisión Sin CSV previo → procesar. CSV previo existe y hubo actividad posterior → re-procesar. Sin actividad reciente → saltear.
ParámetroDescripción
-ConfigPathObligatorio. Path al cliente_config.json.
-DiasUmbralOverride del valor del config. Usar 999 para forzar todos.
-ForceAuthSwitch. Fuerza re-autenticación.
Prerequisitopip install pandas · Los CSVs del Script B deben estar en la carpeta configurada.
ParámetroDescripción
--configPath al cliente_config.json.
--sitioSolo para generar_informes_sitio.py. Nombre exacto de un sitio para generar solo ese informe.
